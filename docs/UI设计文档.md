# 新增记录页面 - UI设计文档

## 1. 页面概览

### 1.1 功能描述

该页面用于用户新增收入记录，包含收入分类选择、金额输入、详细信息填写等功能。

### 1.2 页面结构

- 顶部导航栏：包含标题、取消按钮、确认按钮
- 分类标签栏：切换不同类型的记录（建议、支出、收入、转账、应收款项、应付款项）
- 分类图标网格：展示收入类别的具体选项
- 金额输入区域：货币选择、金额输入、加减按钮
- 详细信息区域：名称、账户、项目、商家、次数、日期、时间
- 标签显示区域
- 备注输入区域

## 2. 布局设计

### 2.1 整体布局

- 页面采用垂直流式布局，适配移动端屏幕
- 顶部导航栏固定，其他内容可滚动
- 各区域之间有适当的间距，确保界面整洁

### 2.2 响应式设计

- 设计目标：适配320px-414px宽度的移动设备
- 布局策略：
  - 分类图标网格：在小屏幕上显示2列，大屏幕上显示3列
  - 输入区域：宽度自适应屏幕，左右边距固定
  - 按钮和图标：大小随屏幕尺寸适当调整

## 4. 交互设计

### 4.1 分类标签切换

- 点击不同标签时，切换到对应分类
- 激活标签显示蓝色下划线和文字颜色变化
- 分类图标网格内容随标签切换而变化

### 4.2 分类图标选择

- 点击图标时，图标背景色变为浅蓝色（#E6F7FF）
- 同时更新选中状态

### 4.3 金额输入

- 点击金额输入框弹出数字键盘
- 点击加减按钮调整金额
- 支持直接输入数字

### 4.4 日期/时间选择

- 点击日期或时间字段弹出日期/时间选择器
- 选择后自动更新显示

### 4.5 保存操作

- 点击右上角确认按钮，验证必填项
- 验证通过后提交数据
- 提交成功后返回上一页

### 4.6 取消操作

- 点击左上角取消按钮，弹出确认对话框
- 确认后返回上一页，不保存当前输入

## 5. 实现注意事项

### 5.1 技术栈选择

- **前端框架**：React Native 或 Flutter（移动端应用）
- **状态管理**：使用 useState 或类似状态管理工具
- **表单处理**：使用表单验证库
- **图标库**：使用 Material Icons 或自定义图标

### 5.2 性能优化

- 图标使用矢量图标，减少资源大小
- 输入框使用防抖处理，优化输入性能
- 分类切换时使用动画过渡，提升用户体验

### 5.3 兼容性考虑

- 适配不同屏幕尺寸和方向
- 支持深色模式
- 考虑不同设备的输入方式（触摸、键盘）

### 5.4 无障碍设计

- 为所有可交互元素添加适当的无障碍标签
- 支持屏幕阅读器
- 确保足够的色彩对比度

4. **用户体验**：添加适当的动画和过渡效果
5. **响应式布局**：确保在不同设备上的良好显示
